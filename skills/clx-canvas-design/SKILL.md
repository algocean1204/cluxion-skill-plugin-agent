# Clx Canvas Design

Create one deliberate static composition, not a decorated document. The default path is fully offline: use local fonts and drawing/export tools already available in the workspace, and never download a font, image, or package without explicit approval.

## Workflow

1. Lock the medium, dimensions, required copy, audience, and output format.
2. Write a short visual direction covering form, space, palette, type, and one signature device. Do not imitate a living artist or a protected work.
3. Build the composition with sparse, real copy. Use hierarchy and negative space instead of repeated cards, decorative labels, or generic gradients.
4. Export PNG or PDF with local tools. If a requested exporter is unavailable offline, preserve an editable SVG and state the limitation; do not silently use a network service.
5. Inspect the rendered artifact at full size. Text, graphics, crop, and margins must stay inside the canvas with accidental overlap and clipping at zero.

## Quality gate

- Respect supplied brand/reference constraints before any aesthetic choice.
- Keep text essential; correct spelling and line breaks before export.
- Use one coherent grid and spacing rhythm. Refine existing elements before adding decoration.
- Produce original work and record any supplied third-party asset attribution.

Read `references/workflow.md` only when the artifact needs a multi-pass critique or print/export checklist.
